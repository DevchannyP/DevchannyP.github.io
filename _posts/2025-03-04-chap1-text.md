# CHAP 1. 본문

### 📌 1단계: 자바 개요 및 설치

이 단계에서는 자바가 무엇인지, 그리고 설치 방법을 설명합니다.

**🔹 자바란?**

자바(Java)는 다양한 플랫폼에서 실행될 수 있는 강력한 프로그래밍 언어입니다.

**🔹 자바의 특징**

✔️ 플랫폼 독립성 – 한 번 작성하면 어디서든 실행 가능

✔️ 객체지향 프로그래밍 – 코드 재사용 및 유지보수 용이

✔️ 자동 메모리 관리 – 가비지 컬렉터(GC)가 메모리를 자동 정리

**🔹 자바 설치 방법**

1️⃣ [오라클 JDK 다운로드](https://www.oracle.com/java/technologies/javase-downloads.html)

2️⃣ 설치 후 환경변수(`JAVA_HOME`, `Path`) 설정

3️⃣ 터미널에서 `java -version` 및 `javac -version`으로 확인

---

### 📌 2단계: 환경변수란? 그리고 왜 설정해야 할까?

이제 **환경변수(Environment Variable)**에 대해 배워볼 거예요! 🚀

환경변수를 설정하면 **어디서든 자바를 실행할 수 있도록** 해줘요.

---

## 🌍 **1. 환경변수란?**

➡️ 환경변수는 **운영체제(Windows, Mac, Linux)가 프로그램을 쉽게 찾을 수 있도록 돕는 설정값**이에요.

💡 **쉽게 설명하면?**

📞 **전화번호부**라고 생각하면 돼요!

- 친구한테 전화하려면 **이름(환경변수)**만 입력하면 자동으로 전화번호(경로)를 찾아주죠?
- 마찬가지로 **환경변수에 자바 경로를 등록하면**, 운영체제가 자바 프로그램(`javac`, `java`)을 자동으로 찾아 실행할 수 있어요!

---

## 🛠 **2. 환경변수를 설정하지 않으면?**

만약 환경변수를 설정하지 않고 `javac`를 실행하면 이런 오류가 발생해요!

```bash
bash
```

😥 **왜 이런 오류가 날까?**

운영체제가 **자바 컴파일러(`javac`)가 어디 있는지 모르기 때문이에요!**


### 📌 4단계: **변수(Variable)와 자료형(Data Type) 이해하기**

이제 자바에서 데이터를 저장하는 **변수(Variable)와 자료형(Data Type)**에 대해 배워볼 거예요! 🚀

변수는 **값을 저장하는 상자**, 자료형은 **어떤 종류의 데이터를 저장할지 정하는 규칙**이에요.

---

## 🎁 **1. 변수란?**

➡ **변수(Variable)**란 값을 저장할 수 있는 **메모리 공간(상자)**이에요.

➡ 프로그래밍에서 데이터를 저장하고 사용하려면 변수가 필요해요.

💡 **쉽게 설명하면?**

- 변수를 **"물건을 담는 상자"**라고 생각하면 돼요.
- *이름표(변수명)**를 붙여야 어떤 값이 들어 있는지 알 수 있어요!
- 숫자, 글자, 참/거짓 값 등 **다양한 종류의 데이터를 변수에 저장할 수 있어요.**

---

## 🏗 **2. 변수 선언과 사용 방법**

변수를 사용하려면 **1️⃣ 선언 → 2️⃣ 값 저장 → 3️⃣ 출력**하는 과정이 필요해요!

```java
java
복사편집
// 1️⃣ 변수 선언
int number;

// 2️⃣ 변수에 값 저장

// 3️⃣ 변수 값 출력

```

📌 **출력 결과:**

```

10

```

---

## 📏 **3. 변수의 종류(자료형, Data Type)**

변수에는 **저장할 데이터의 종류에 따라 자료형(Data Type)이 달라요!**

| 자료형 | 설명 | 예제 |
| --- | --- | --- |
| `int` | 정수(소수점 없는 숫자) | `int age = 25;` |
| `double` | 실수(소수점 있는 숫자) | `double height = 175.5;` |
| `char` | 한 글자(문자 하나) | `char grade = 'A';` |
| `String` | 여러 글자(문자열) | `String name = "홍길동";` |
| `boolean` | 참(`true`) 또는 거짓(`false`) | `boolean isJavaFun = true;` |

💡 **비유하면?**

- `int` → 숫자를 넣는 **숫자 상자**
- `double` → 소수를 넣는 **소수 상자**
- `char` → 한 글자를 넣는 **글자 상자**
- `String` → 여러 글자를 넣는 **문장 상자**
- `boolean` → 참/거짓을 넣는 **스위치(ON/OFF 버튼)**

---

## ✏️ **4. 변수 값 바꾸기 (변수는 한 번에 하나의 값만 저장!)**

변수는 값을 바꿀 수 있어요!

하지만 **한 번에 하나의 값만 저장 가능**하고, 새 값을 넣으면 이전 값은 사라져요.

```java
java
복사편집
int number = 10;
System.out.println(number);  // 10 출력

number = 20;
System.out.println(number);  // 20 출력 (이전 값 10은 사라짐)

```

📌 **출력 결과:**

```
복사편집
10
20

```

✔ **변수는 "값을 저장하는 상자"이므로, 새로운 값을 넣으면 기존 값이 덮어씌워져요!**

### 📌5단계: **연산자(Operators) 이해하기**

자바에서 **연산자(Operators)**는 숫자 계산, 값 비교, 논리 판단 등 다양한 작업을 수행할 수 있도록 도와주는 도구예요!

오늘은 **사칙연산, 비교 연산, 논리 연산**을 쉽게 배워볼 거예요! 🚀

## ➕ **1. 산술 연산자 (사칙연산)**

➡️ **숫자를 계산할 때 사용하는 연산자**예요.

➡️ 우리가 수학 시간에 배운 덧셈, 뺄셈, 곱셈, 나눗셈을 코드로 표현할 수 있어요!

✅ **자바의 기본 연산자**

| 연산자 | 설명 | 예제 (`a = 10, b = 3`) | 결과 |
| --- | --- | --- | --- |
| `+` | 더하기 | `a + b` | `13` |
| `-` | 빼기 | `a - b` | `7` |
| `*` | 곱하기 | `a * b` | `30` |
| `/` | 나누기(몫) | `a / b` | `3` (정수 나눗셈) |
| `%` | 나머지 | `a % b` | `1` |

---

### ✏️ **코드 예제**

```java
java
복사편집
public class ArithmeticOperators {
    public static void main(String[] args) {
        int a = 10;
        int b = 3;

        System.out.println("덧셈: " + (a + b));  // 13
        System.out.println("뺄셈: " + (a - b));  // 7
        System.out.println("곱셈: " + (a * b));  // 30
        System.out.println("나눗셈: " + (a / b));  // 3 (정수라서 몫만 나옴)
        System.out.println("나머지: " + (a % b));  // 1
    }
}

```

📌 **실수(소수점 있는 숫자) 연산을 하려면 `double`을 사용해야 해요!**

```java
java
복사편집
double x = 10.0;
double y = 3.0;
System.out.println(x / y);  // 3.333333

```

## 🔍 **2. 비교 연산자 (크기 비교하기)**

➡️ **두 개의 값을 비교하는 연산자**예요.

➡️ 비교 연산자는 결과가 **참(true) 또는 거짓(false)**으로 나와요!

✅ **자바의 비교 연산자**

| 연산자 | 설명 | 예제 (`a = 10, b = 3`) | 결과 |
| --- | --- | --- | --- |
| `==` | 같다 | `a == b` | `false` |
| `!=` | 같지 않다 | `a != b` | `true` |
| `>` | 크다 | `a > b` | `true` |
| `<` | 작다 | `a < b` | `false` |
| `>=` | 크거나 같다 | `a >= b` | `true` |
| `<=` | 작거나 같다 | `a <= b` | `false` |

---

### ✏️ **코드 예제**

```java
java
복사편집
public class ComparisonOperators {
    public static void main(String[] args) {
        int a = 10;
        int b = 3;

        System.out.println(a == b);  // false
        System.out.println(a != b);  // true
        System.out.println(a > b);   // true
        System.out.println(a < b);   // false
        System.out.println(a >= b);  // true
        System.out.println(a <= b);  // false
    }
}

```

📌 **비교 연산자는 `boolean` 값을 반환해요!**

➡ **`true`(참) 또는 `false`(거짓)으로 결과가 나와요.**

## 🤔 **3. 논리 연산자 (AND, OR, NOT)**

➡ **논리적인 조건을 연결하거나 부정하는 연산자**예요.

➡ **참(`true`)과 거짓(`false`)을 조합할 때 사용해요!**

✅ **자바의 논리 연산자**

| 연산자 | 설명 | 예제 (`a = 10, b = 3`) | 결과 |
| --- | --- | --- | --- |
| `&&` | AND (둘 다 참이면 참) | `(a > 5 && b > 5)` | `false` |
| `||` | OR (하나라도 참이면 참) | ``(a > 5 || b > 5)` | `true` |
| `!` | NOT (반대로 바꿈) | `!(a > 5)` | `false` |

---

### ✏️ **코드 예제**

```java
java
복사편집
public class LogicalOperators {
    public static void main(String[] args) {
        int a = 10;
        int b = 3;

        System.out.println(a > 5 && b > 5);  // false (b > 5가 거짓)
        System.out.println(a > 5 || b > 5);  // true (a > 5가 참)
        System.out.println(!(a > 5));        // false (a > 5가 참이므로 반대는 거짓)
    }
}

```

📌 **논리 연산자는 여러 개의 조건을 한 번에 확인할 때 유용해요!**

```java
java
복사편집
int age = 20;
boolean isStudent = false;

if (age >= 18 && !isStudent) {
    System.out.println("성인이고 학생이 아닙니다!");
}

```

✔ **`&&`는 둘 다 참이어야 참!**

✔ **`||`는 하나라도 참이면 참!**

✔ **`!`는 참이면 거짓, 거짓이면 참!**

## 🎯 **4. 연산자 우선순위**

자바의 연산자는 **수학처럼 우선순위가 있어요!**

1️⃣ 먼저 **괄호 `()`** 안의 연산을 계산

2️⃣ 곱하기(`*`), 나누기(`/`), 나머지(`%`) 연산이 덧셈(`+`), 뺄셈(`-`)보다 먼저 실행

3️⃣ 비교 연산자(`>`, `<`, `==`)가 산술 연산자보다 늦게 실행

4️⃣ 논리 연산자 `&&`가 `||`보다 먼저 실행

📌 **우선순위 예제**

```java
java
복사편집
System.out.println(10 + 2 * 3);   // 16 (곱하기가 먼저)
System.out.println((10 + 2) * 3); // 36 (괄호가 먼저)
System.out.println(10 > 5 && 2 < 3); // true
System.out.println(10 > 5 || 2 > 3); // true

```

✔ **괄호를 활용하면 연산 순서를 명확하게 정할 수 있어요!**

## 🎯 **1. 메서드(Method)란?**

➡ **메서드란?** 특정 기능을 수행하는 **코드 블록(함수)**

➡ 한 번 만들어 놓으면 **필요할 때마다 호출해서 재사용 가능!**

💡 **비유하면?**

- **세탁기** 버튼을 누르면 자동으로 빨래가 되죠?
- *메서드도 비슷하게, 특정 작업을 실행하는 "기능"**이라고 생각하면 돼요!

---

## 🏗 **2. 메서드 만들기 (선언 & 호출)**

### ✏️ **(1) 기본 메서드 예제**

```java
java
복사편집
public class MethodExample {
    // 1️⃣ 메서드 선언
    static void sayHello() {
        System.out.println("안녕하세요!");
    }

    public static void main(String[] args) {
        // 2️⃣ 메서드 호출 (사용)
        sayHello();
    }
}

```

📌 **출력 결과:**

```
복사편집
안녕하세요!

```

✔ **메서드를 사용하면 같은 코드를 여러 번 반복하지 않아도 돼요!**

---

### ✏️ **(2) 매개변수(입력값) 있는 메서드**

```java
java
복사편집
static void greet(String name) {
    System.out.println("안녕하세요, " + name + "님!");
}

public static void main(String[] args) {
    greet("철수");
    greet("영희");
}

```

📌 **출력 결과:**

```
복사편집
안녕하세요, 철수님!
안녕하세요, 영희님!

```

✔ **매개변수(parameter)** → 메서드가 받을 입력값을 의미!

✔ `greet("철수")` → `"철수"`라는 값을 전달해서 실행

---

### ✏️ **(3) 반환값(Return) 있는 메서드**

➡ 메서드는 **결과값을 반환(return)할 수도 있어요!**

```java
java
복사편집
static int add(int a, int b) {
    return a + b;  // 결과 반환
}

public static void main(String[] args) {
    int sum = add(5, 3);
    System.out.println("결과: " + sum);
}

```

📌 **출력 결과:**

```
makefile

결과: 8

```

✔ **`return` 키워드를 사용해 결과값을 돌려줌!**

✔ `add(5, 3)`을 호출하면 **8이 반환되어 `sum`에 저장**됨

## 👤 **3. 객체(Object)란? (자바의 핵심 OOP 개념!)**

➡ **자바는 객체지향 프로그래밍(OOP) 언어**

➡ **객체(Object)**는 **속성(변수)과 기능(메서드)을 포함하는 데이터 구조**

💡 **비유하면?**

- *자동차(객체)**는 **속성(색깔, 브랜드)과 기능(달리기, 멈추기)**를 가지고 있어요.
- **자바의 객체**도 비슷하게 **속성(변수)과 기능(메서드)**로 구성돼요!

---

### ✏️ **(1) 클래스(Class)와 객체(Object) 만들기**

```java
java
복사편집
// 1️⃣ 클래스 선언 (설계도 만들기)
class Car {
    String color;  // 속성 (변수)

    void drive() {  // 기능 (메서드)
        System.out.println(color + " 자동차가 달립니다!");
    }
}

public class ObjectExample {
    public static void main(String[] args) {
        // 2️⃣ 객체 생성 (실제 자동차 만들기)
        Car myCar = new Car();

        // 3️⃣ 속성 값 설정
        myCar.color = "빨간색";

        // 4️⃣ 메서드 호출
        myCar.drive();
    }
}

```

📌 **출력 결과:**

```
복사편집
빨간색 자동차가 달립니다!

```

✔ **클래스(Class)** → 객체를 만들기 위한 **설계도**

✔ **객체(Object)** → 클래스를 기반으로 만들어진 **실제 데이터**

---

## 🎯 **5. 클래스 vs. 객체 vs. 메서드 정리!**

| 개념 | 설명 | 예제 |
| --- | --- | --- |
| **클래스(Class)** | 객체를 만들기 위한 **설계도** | `class Car {}` |
| **객체(Object)** | 클래스를 기반으로 만들어진 **실제 데이터** | `Car myCar = new Car();` |
| **속성(변수)** | 객체가 가지고 있는 값 | `myCar.color = "빨간색";` |
| **메서드(Method)** | 객체가 할 수 있는 기능 | `myCar.drive();` |
| **생성자(Constructor)** | 객체 생성 시 실행되는 초기화 메서드 | `Person(String name) {}` |

✔ **클래스**는 **설계도**

✔ **객체**는 **실제 자동차**

✔ **메서드**는 **자동차가 움직이는 기능**

## 🎯**6. 객체지향 프로그래밍(OOP)의 4가지 특징**

**1️⃣ 캡슐화 (Encapsulation)**

➡ **객체 내부의 데이터를 보호하는 개념**

➡ 예제: `private` 키워드를 사용해 외부에서 직접 접근 못 하게 막음!

**2️⃣ 상속 (Inheritance)**

➡ **부모 클래스의 기능을 자식 클래스가 물려받는 개념**

➡ 예제: `class Dog extends Animal {}`

**3️⃣ 다형성 (Polymorphism)**

➡ **같은 메서드를 다른 방식으로 동작하게 만드는 개념**

➡ 예제: `오버라이딩(Override)`과 `오버로딩(Overload)`

**4️⃣ 추상화 (Abstraction)**

➡ **필요한 부분만 보여주고, 불필요한 부분은 숨기는 개념**

➡ 예제: `abstract class`와 `interface` 사용

## 🎯 **1. 캡슐화(Encapsulation) - 데이터를 보호하는 기술**

➡ **클래스 내부의 데이터를 숨기고(private), 메서드를 통해서만 접근하게 하는 방식**

➡ **잘못된 값이 저장되는 것을 막고, 보안을 강화할 수 있어요!**

💡 **비유하면?**

- **ATM 기계**를 사용할 때, 내부 기계를 직접 조작하지 않고 버튼만 누르죠?
- **캡슐화도 마찬가지로, 클래스 내부의 데이터를 직접 수정하지 않고 메서드를 통해 조작해요!**

---

### ✏️ **(1) 캡슐화 예제 (getter & setter 사용하기)**

```java
java
복사편집
class Person {
    private String name;  // ❌ 직접 접근 불가능

    // 값을 설정하는 메서드 (Setter)
    public void setName(String newName) {
        name = newName;
    }

    // 값을 가져오는 메서드 (Getter)
    public String getName() {
        return name;
    }
}

public class EncapsulationExample {
    public static void main(String[] args) {
        Person p1 = new Person();
        p1.setName("철수");  // 값 설정
        System.out.println(p1.getName());  // 철수 출력
    }
}

```

📌 **출력 결과:**

```
복사편집
철수

```

✔ `private` → 직접 접근 ❌

✔ `getter`, `setter`를 통해 안전하게 값 변경 ✅

## 🔄 **2. 상속(Inheritance) - 코드 재사용하기**

➡ **부모 클래스(기본 기능)를 자식 클래스(추가 기능)가 물려받는 개념**

➡ **공통된 기능을 상속받아 코드 중복을 줄일 수 있어요!**

💡 **비유하면?**

- **부모가 자식에게 DNA(기능)를 물려주는 것과 같아요!**

---

### ✏️ **(1) 상속 기본 예제**

```java
java
복사편집
// 부모 클래스
class Animal {
    void eat() {
        System.out.println("음식을 먹습니다.");
    }
}

// 자식 클래스 (Animal의 기능을 물려받음)
class Dog extends Animal {
    void bark() {
        System.out.println("멍멍!");
    }
}

public class InheritanceExample {
    public static void main(String[] args) {
        Dog myDog = new Dog();
        myDog.eat();  // 부모 클래스 기능 사용 가능!
        myDog.bark(); // 자식 클래스 기능 사용 가능!
    }
}

```

📌 **출력 결과:**

```
복사편집
음식을 먹습니다.
멍멍!

```

✔ **자식 클래스(Dog)는 부모 클래스(Animal)의 기능(eat)을 물려받음**

✔ **`extends` 키워드를 사용해 상속 구현**

## 🔄 **3. 다형성(Polymorphism) - 같은 기능, 다른 동작!**

➡ **같은 이름의 메서드가 상황에 따라 다르게 동작하는 개념**

➡ **코드를 더 유연하게 만들 수 있어요!**

💡 **비유하면?**

- **"운전하다"라는 동작은 자동차, 오토바이, 자전거에서 다르게 동작하죠?**
- **같은 "run()" 메서드가 자동차와 오토바이에서 다르게 동작하는 것과 같아요!**

---

### ✏️ **(1) 메서드 오버라이딩(Override) 예제**

```java
java
복사편집
class Animal {
    void sound() {
        System.out.println("동물이 소리를 냅니다.");
    }
}

class Dog extends Animal {
    @Override
    void sound() {  // 부모 메서드 재정의
        System.out.println("멍멍!");
    }
}

public class PolymorphismExample {
    public static void main(String[] args) {
        Animal myAnimal = new Dog();
        myAnimal.sound();  // 멍멍! 출력
    }
}

```

📌 **출력 결과:**

```
복사편집
멍멍!

```

✔ **부모 클래스의 `sound()`를 자식 클래스에서 재정의(Override) 가능!**

✔ **`@Override` 키워드를 사용하면 오버라이딩을 명확히 표시할 수 있음**

---

### ✏️ **(2) 메서드 오버로딩(Overloading) 예제**

```java
java
복사편집
class MathUtils {
    int add(int a, int b) {
        return a + b;
    }

    // 같은 이름의 메서드, 다른 매개변수
    double add(double a, double b) {
        return a + b;
    }
}

public class OverloadingExample {
    public static void main(String[] args) {
        MathUtils math = new MathUtils();
        System.out.println(math.add(3, 5));      // 8 출력
        System.out.println(math.add(2.5, 3.5));  // 6.0 출력
    }
}

```

📌 **출력 결과:**

```
복사편집
8
6.0

```

✔ **같은 `add()` 메서드를 여러 방식으로 사용 가능!**

## 🔍 **4. 추상화(Abstraction) - 불필요한 정보 숨기기**

➡ **사용자는 내부 동작을 몰라도 기능을 사용할 수 있도록 설계!**

➡ **추상 클래스(Abstract Class)와 인터페이스(Interface)로 구현 가능!**

💡 **비유하면?**

- **자동차 운전할 때 엔진 내부 구조를 몰라도 운전할 수 있죠?**
- **자바의 추상화도 마찬가지로, 내부 구현은 숨기고 필요한 기능만 보여줘요!**

---

### ✏️ **(1) 추상 클래스(Abstract Class) 예제**

```java
java
복사편집
abstract class Animal {
    abstract void makeSound(); // 추상 메서드 (구현 없음)

    void eat() { // 일반 메서드 (구현 있음)
        System.out.println("음식을 먹습니다.");
    }
}

class Dog extends Animal {
    @Override
    void makeSound() {
        System.out.println("멍멍!");
    }
}

public class AbstractionExample {
    public static void main(String[] args) {
        Dog myDog = new Dog();
        myDog.makeSound();  // 멍멍! 출력
        myDog.eat();  // 음식 먹기 출력
    }
}

```

📌 **출력 결과:**

```
복사편집
멍멍!
음식을 먹습니다.

```

✔ **`abstract` 클래스는 직접 객체 생성 ❌ (반드시 상속해서 사용)**

✔ **추상 메서드(구현 없음)는 반드시 자식 클래스에서 구현해야 함!**

## 🛠 **3. try-catch로 예외 처리하기**

📌 **예외가 발생할 가능성이 있는 코드를 `try` 블록 안에 넣고, 오류 발생 시 `catch` 블록에서 처리해요!**

## 🎯 **1. 멀티스레드란? (Single-Thread vs. Multi-Thread)**

💡 **비유하면?**

- **싱글 스레드(Single-Thread)** → 한 사람이 한 가지 일만 하는 경우
- **멀티스레드(Multi-Thread)** → 여러 사람이 동시에 여러 일을 하는 경우

➡ **자바의 기본 실행 방식은 "싱글 스레드"** (한 번에 한 가지 작업)

## 🏗 **2. 스레드(Thread) 만들기**

➡ **자바에서 스레드를 만드는 방법은 2가지!**

✔ `Thread` 클래스를 상속하는 방법

✔ `Runnable` 인터페이스를 구현하는 방법

### ✏️ **(1) Thread 클래스 상속하기**

```java

복사편집

            System.out.println("쓰레드 실행 중: " + i);

                e.printStackTrace();
            }
        }
    }
}

public class ThreadExample {
    public static void main(String[] args) {
        MyThread t1 = new MyThread();  // 스레드 객체 생성

}
```

✔ **`Thread` 클래스를 상속하고 `run()` 메서드를 오버라이드**

✔ **`start()` 메서드를 호출해야 스레드가 실행됨! (`run()` 직접 호출 ❌)**

### ✏️ **(2) Runnable 인터페이스 구현하기**


✔ `Runnable` 인터페이스를 구현하고 `run()` 메서드 작성

✔ `new Thread(new MyRunnable())` → `Thread` 객체에 전달해서 실행

➡ **두 방법의 차이?**

✔ `Thread` 클래스를 상속하면 다른 클래스를 상속할 수 없음 ❌

✔ `Runnable` 인터페이스는 다른 클래스를 상속하면서 스레드 기능 추가 가능 ✅

```java
}

class Task2 extends Thread {
    public void run() {
        for (int i = 1; i <= 3; i++) {
            System.out.println("Task 2 실행 중: " + i);
            try { Thread.sleep(1000); } catch (InterruptedException e) {}
        }
    }
}

public class MultiThreadExample {
    public static void main(String[] args) {
        Task1 t1 = new Task1();
        Task2 t2 = new Task2();

        t1.start();  // 스레드 1 실행
        t2.start();  // 스레드 2 실행
    }
}

```

📌 **출력 결과 (동시에 실행됨!)**

```
arduino
복사편집
Task 1 실행 중: 1
Task 2 실행 중: 1
Task 1 실행 중: 2
Task 2 실행 중: 2
Task 1 실행 중: 3
Task 2 실행 중: 3

```

✔ **`start()`를 여러 번 호출하면 여러 개의 스레드가 동시에 실행됨!**

✔ **각 스레드는 독립적으로 실행되므로 실행 순서는 매번 다를 수 있음!**

## 🔁 **4. 스레드 동기화 (synchronized) - 데이터 충돌 방지**

➡ **여러 개의 스레드가 동시에 같은 데이터에 접근하면 문제가 발생할 수 있어요!**

➡ **`synchronized` 키워드를 사용하면 하나의 스레드만 접근 가능하도록 설정할 수 있어요!**

---

### ✏️ **(1) 동기화 문제 발생 (충돌 발생!)**

```java
java
복사편집
class BankAccount {
    private int balance = 1000;

    void withdraw(int amount) {
        if (balance >= amount) {
            System.out.println(Thread.currentThread().getName() + " 출금 중...");
            balance -= amount;
            System.out.println("잔액: " + balance);
        } else {
            System.out.println("잔액 부족!");
        }
    }
}

public class BankExample {
    public static void main(String[] args) {
        BankAccount account = new BankAccount();

        Thread t1 = new Thread(() -> account.withdraw(800), "사용자1");
        Thread t2 = new Thread(() -> account.withdraw(800), "사용자2");

        t1.start();
        t2.start();
    }
}

```

📌 **출력 결과 (데이터 충돌 발생!)**

```
makefile
복사편집
사용자1 출금 중...
사용자2 출금 중...
잔액: 200
잔액: -600 (오류 발생!)

```

✔ **두 스레드가 동시에 출금 요청을 해서 잔액이 음수가 됨! (데이터 충돌)**

---

### ✏️ **(2) synchronized 키워드 사용 (충돌 방지!)**

```java
java
복사편집
class BankAccount {
    private int balance = 1000;

    synchronized void withdraw(int amount) {  // 🔒 한 번에 한 스레드만 실행 가능
        if (balance >= amount) {
            System.out.println(Thread.currentThread().getName() + " 출금 중...");
            balance -= amount;
            System.out.println("잔액: " + balance);
        } else {
            System.out.println("잔액 부족!");
        }
    }
}

```

📌 **출력 결과 (동기화 성공!)**

```
makefile
복사편집
사용자1 출금 중...
잔액: 200
잔액 부족!

```

✔ **`synchronized`를 사용하면 한 번에 한 스레드만 실행 가능!**

✔ **데이터 충돌 없이 안전하게 실행됨!**

## 📌 **5. 멀티스레드 핵심 정리!**

| 개념 | 설명 | 예제 |
| --- | --- | --- |
| **Thread 클래스** | `Thread`를 상속해서 스레드 만들기 | `class MyThread extends Thread {}` |
| **Runnable 인터페이스** | `Runnable` 구현해서 스레드 만들기 | `class MyRunnable implements Runnable {}` |
| **start()** | 스레드 실행 | `t1.start();` |
| **sleep()** | 일정 시간 멈춤 | `Thread.sleep(1000);` |
| **synchronized** | 데이터 충돌 방지 | `synchronized void method() {}` |
